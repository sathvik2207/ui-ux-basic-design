# Color Palette

## Project Title

**TaskFlow – A Smart Task Management Mobile Application**

---

# Introduction

The TaskFlow color palette establishes a consistent visual identity while supporting usability, accessibility, and emotional engagement. Each color has a defined purpose to help users quickly recognize actions, statuses, and interface elements.

The palette is designed with accessibility in mind, ensuring sufficient contrast for readability and supporting both light and dark themes.

---

# Design Goals

The color system aims to:

* Create a modern and professional appearance.
* Improve readability and navigation.
* Reinforce important actions through color.
* Maintain visual consistency across screens.
* Support accessibility standards.
* Provide a pleasant experience in both light and dark modes.

---

# Color Usage Guidelines

## Use Primary Blue For

* Login button
* Save button
* Active navigation item
* Links
* Floating Action Button (FAB)

---

## Use Green For

* Completed tasks
* Success messages
* Positive statistics

---

## Use Red For

* Delete buttons
* Error alerts
* High-priority tasks
* Validation messages

---

## Use Orange For

* Warnings
* Pending tasks
* Due soon reminders

---

## Use Gray For

* Secondary text
* Disabled buttons
* Borders
* Dividers
* Placeholder text

---

# Accessibility Considerations

The palette follows accessibility best practices by:

* Providing sufficient contrast between text and backgrounds.
* Avoiding the use of color as the sole indicator of meaning.
* Pairing colors with icons or labels for important statuses.
* Ensuring readability in both light and dark themes.
* Maintaining consistent semantic color usage throughout the application.
