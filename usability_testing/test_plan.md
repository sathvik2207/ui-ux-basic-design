# Introduction

Usability testing evaluates how effectively users can interact with the TaskFlow prototype to complete common tasks. The objective is to identify usability issues, validate design decisions, and gather actionable feedback before development.

The testing focuses on ease of use, navigation, task completion, accessibility, and overall user satisfaction.

---

# Testing Objectives

The usability testing aims to:

* Validate the application's navigation.
* Evaluate task creation efficiency.
* Measure ease of learning.
* Identify user pain points.
* Assess overall satisfaction.
* Verify accessibility and readability.
* Discover opportunities for design improvements.

---

# Research Questions

The study seeks answers to the following questions:

1. Can users complete essential tasks without assistance?
2. Is the navigation intuitive?
3. How quickly can users create a new task?
4. Are reminders and notifications easy to configure?
5. Can users locate completed and pending tasks easily?
6. Does the interface feel simple and uncluttered?
7. Are users satisfied with the overall experience?

# Test Scenarios

## Scenario 1 – Login

Objective:

Log in to the application.

Expected Outcome:

Participant reaches the Home Dashboard successfully.

---

## Scenario 2 – Create a Task

Objective:

Create a task named:

**"Complete UI Assignment"**

Requirements:

* Category: Study
* Priority: High
* Due Date: Tomorrow
* Reminder: 6:00 PM

Expected Outcome:

Task is created successfully.

---

## Scenario 3 – Edit a Task

Objective:

Update the reminder time for the previously created task.

Expected Outcome:

Changes are saved successfully.

---

## Scenario 4 – Complete a Task

Objective:

Mark the task as completed.

Expected Outcome:

Task moves to the completed section and statistics update accordingly.

---

## Scenario 5 – Search

Objective:

Find the task using the search feature.

Expected Outcome:

Relevant task appears in search results.

---

## Scenario 6 – Calendar

Objective:

Navigate to tomorrow's date and view scheduled tasks.

Expected Outcome:

The correct tasks are displayed.

---

## Scenario 7 – Profile

Objective:

Switch the application to Dark Mode.

Expected Outcome:

Theme changes successfully.

# Observation Metrics

During testing, record:

* Time taken to complete each task.
* Number of clicks or taps.
* Navigation errors.
* User hesitation.
* Questions asked by participants.
* Confusion points.
* Verbal comments.

---

# Success Criteria

A task is considered successful if the participant:

* Completes it without assistance.
* Makes no critical errors.
* Finishes within the expected time.
* Expresses confidence in the process.

# Post-Test Questionnaire

Participants rate the following statements on a scale of **1 (Strongly Disagree)** to **5 (Strongly Agree)**:

1. The application was easy to use.
2. Navigation was intuitive.
3. Creating tasks was straightforward.
4. The interface was visually appealing.
5. I could easily find important information.
6. I would use this application regularly.
7. I would recommend this application to others.

---

# Open-Ended Questions

Participants are asked:

* What did you like most about TaskFlow?
* What was the most confusing part?
* Which feature was most useful?
* Which feature needs improvement?
* Was anything missing?
* Do you have any additional suggestions?

---

# Risks and Mitigation

| Risk                                  | Mitigation                                      |
| ------------------------------------- | ----------------------------------------------- |
| Participant unfamiliar with prototype | Provide a brief introduction before testing     |
| Internet connectivity issues          | Use a downloaded prototype or stable connection |
| Time constraints                      | Limit each session to 45 minutes                |
| Biased feedback                       | Ask neutral, open-ended questions               |
